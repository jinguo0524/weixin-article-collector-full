# 微信公众号文章搜集规则

## 目标公众号（固定列表）

在 `config.js` 中配置：

1. 公众号A
2. 公众号B
3. 公众号C

## 数据存储策略

### 底量内容（自动维护）

- 每个公众号在 SQLite 库中保留最新 **20 篇**文章（含正文）
- 每次同步后自动清理超出 20 篇的旧文章
- 数据库路径：`data/weixin.db`

### 增量更新

- 每次运行只抓取**昨天和今天**两天的新文章
- 新文章自动入库（去重），并立即抓取正文内容
- 正文抓取间隔：800ms/篇，避免触发反爬

## 正文提取

微信文章正文嵌入在页面 JS 变量 `content_noencode` 中（当前格式），提取时需处理 `\x0a`（换行）、`\x22`（双引号）等转义。回退方案：旧格式使用 cheerio 提取 `#js_content` 元素。

## 输出规则

- 按**日期分目录**：`output/YYYY-MM-DD/`
- 每篇文章独立 `.md` 文件（含标题、来源、时间、原文链接、正文）
- 每个日期目录下生成 `_汇总.md`（按公众号分组，含链接和发布时间）
- 输出范围：昨天+今天的文章
- 若某公众号当日无更新，标注"今日无更新"

## 日常操作命令

| 命令 | 用途 |
|------|------|
| `npm run login` | 首次使用：扫码登录微信读书，生成 auth.json |
| `npm run setup` | 首次使用：配置目标公众号（需提供每号一篇任意文章链接） |
| `npm run sync` | 核心命令：增量同步昨天+今天的文章列表+正文 |
| `npm run output` | 将昨天+今天的文章按日期输出到 output/ 目录 |
| `npm run run` | 一键执行：sync + output |
| `npm run fetch-content` | 补充抓取：对库中没有正文的文章重新抓取正文 |

## 日常使用流程

```bash
# 每天执行一次即可
npm run run
```

首次使用需要先 `npm run login` 扫码登录，再 `npm run setup -- --add "公众号名" "文章链接"` 逐个添加公众号。

## 技术架构

- **API 代理**：weread.111965.xyz（微信读书 API 代理，稳定性依赖重试机制）
- **存储**：Node.js 内置 SQLite（`node:sqlite`），需 `--experimental-sqlite` 标志
- **登录**：微信扫码登录，token 有效期约 1-2 小时，存于 `auth.json`
- **重试**：API 返回空数组时自动重试最多 5 次（每次间隔 400ms）
